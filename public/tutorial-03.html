Install a Third-Party Library (express)

A Node.js web server is just a Node.js script that listens for HTTP requests. Instead of learning the HTTP protocol, let's use someone else's HTTP library.

We'll use express (which uses connect, which uses the built-in http library (Node.js libraries often compose well)).

  1. You usually add third-party libraries to your project with the npm command. Install Express in the node-course directory like this: 

    $ npm install express
[lots of output, ending with...]
express@3.2.0 ./node_modules/express 
├── methods@0.0.1
├── fresh@0.1.0
├── cookie-signature@1.0.1
├── range-parser@0.0.4
├── buffer-crc32@0.2.1
├── cookie@0.0.5
├── commander@0.6.1
├── debug@0.7.2
├── mkdirp@0.3.5
├── send@0.1.0 (mime@1.2.6)
└── connect@2.7.6 (pause@0.0.1, buffer-crc32@0.1.1, bytes@0.2.0, formidable@1.0.11, qs@0.5.1)

  2. I highlighted the version of Express that was installed because you want to record it in package.json. When you deploy a Node.js server, package.json will usually be used to install your server's dependencies, so it's good to keep it up-to-date.

    Modify the package.json file to look like this: 

    {
    "dependencies" : {
        "express" : "3.2.x"
    }
}

    We used "3.2.x" instead of "3.2.0" so that if 3.2.1 happens to fix a bug, it will automatically be used instead. (If you like, read more about semantic versioning and npm dependency best practices.)

    JSON is a simple syntax for writing objects with keys and values, just like in JavaScript. It's a standard syntax for transmitting objects as strings and we'll be using it a lot.
